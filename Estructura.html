Estructura y funcionamiento de una API

Estructura de una API en WEB

Las APIs web más comunes suelen tener los siguientes componentes

  1. Endpoint (Punto Final):

    Es la URL específica que representa un recurso al que se puede acceder. Por ejemplo, en una API de usuarios, el endpoint podría ser https://api.ejemplo.com/usuarios .

  2. Métodos HTTP:

    Definen la acción que el cliente desea realizar sobre el recurso. Los más comunes son:

    • GET: Obtener o leer datos.
    • POST: Crear un nuevo recurso.
    • PUT/PATCH: Actualizar un recurso existente.
    • DELETE: Eliminar un recurso.

  3. Parámetros

    Información adicional para especificar o filtrar la solicitud.

    • Parámetros de Ruta: Parte de la URL (ej: /usuarios/123).
    • Parámetros de Consulta (Query): Después del ? (ej: /libros?autor=garcia).
    • Cabeceras (Headers): Metadatos, como claves de autenticación o formato de contenido (Content-Type).

  4. Cuerpo (Body)

    Los datos que se envían al servidor en las solicitudes POST o PUT, a menudo formateados en JSON.

Funcionamiento (Ciclo Solicitud-Respuesta)

La comunicación se basa en un ciclo constante entre un Cliente (quien solicita) y un Servidor (quien responde).

  1. Solicitud (Request)

    El cliente envía una solicitud al*Endpoint con un Método HTTP específico, incluyendo Cabeceras y, si aplica, un Cuerpo con datos.

  2. Procesamiento

    La API en el servidor recibe, interpreta la solicitud y ejecuta la lógica necesaria (ej: consultar la base de datos).

  3. Respuesta (Response)

    El servidor genera una respuesta que incluye un Código de Estado HTTP y el Cuerpo de la Respuesta con los datos solicitados (generalmente en JSON) o un mensaje de error.

  4. Entrega

    La API envía la respuesta de vuelta al cliente para su uso o manejo del error.
